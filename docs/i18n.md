# i18n and translations

**What you'll learn:** How Django's translation system works inside Reflex event handlers, and how the active language is chosen for each event.

**When you need this:**

- Your UI strings live in `.po` files and should follow the user's language.
- You need RTL layout hints for Arabic, Hebrew, or similar locales.

Django's `gettext` machinery works in Reflex handlers the same way it works in HTTP views. The event bridge activates the correct language before your handler runs. See [Custom middleware in events](django_middleware_to_reflex.md) for the underlying pipeline.

---

## How the language is chosen

For each Reflex event, the bridge picks a language in this order:

1. Session value (`django_language` session key)
2. `django_language` cookie
3. `Accept-Language` from the WebSocket handshake
4. `LANGUAGE_CODE` in `settings.py`

Then it calls `translation.activate(lang)`. Handlers can use `_()` normally. `DjangoUserState.language` mirrors the active code when `REFLEX_DJANGO_MIRROR_LANGUAGE` is `True` (the default).

## Django setup

```python
USE_I18N = True
USE_TZ = True

LANGUAGE_CODE = "en"

LANGUAGES = [
    ("en", "English"),
    ("ar", "Arabic"),
    ("fr", "French"),
]

LOCALE_PATHS = [BASE_DIR / "locale"]

MIDDLEWARE = [
    # ...
    "django.middleware.locale.LocaleMiddleware",
    # ...
    "reflex_django.streaming_middleware.AsyncStreamingMiddleware",
]
```

Extract and compile messages:

```bash
mkdir -p locale/ar locale/fr
python manage.py makemessages -l ar
python manage.py makemessages -l fr
# edit locale/<lang>/LC_MESSAGES/django.po
python manage.py compilemessages
```

## Strings in components

Use `gettext_lazy` for text inside component trees (compiled ahead of time):

```python
from django.utils.translation import gettext_lazy as _
import reflex as rx
from reflex_django.pages.decorators import page


@page(route="/", title=_("Home"))
def index() -> rx.Component:
    return rx.vstack(
        rx.heading(_("Welcome")),
        rx.text(_("Browse our catalog and sign in to place orders.")),
    )
```

## Strings in event handlers

Use plain `gettext` for toasts and messages created at event time:

```python
from django.utils.translation import gettext as _
from reflex_django.states import AppState


class CheckoutState(AppState):
    @rx.event
    async def submit(self):
        try:
            ...
            return rx.toast.success(_("Order placed successfully."))
        except Exception:
            return rx.toast.error(_("Something went wrong. Please try again."))
```

## Language in the UI

```python
from reflex_django.states import DjangoUserState


def language_badge():
    return rx.text("Language: ", DjangoUserState.language)


def app_shell(content):
    return rx.box(
        content,
        style={"direction": rx.cond(DjangoUserState.language_bidi, "rtl", "ltr")},
    )
```

`language_bidi` is `True` for RTL locales. Set `REFLEX_DJANGO_MIRROR_LANGUAGE = False` if you do not need these vars.

## Switching language at runtime

Mount Django's `set_language` view:

```python
# config/urls.py
from django.urls import path
from django.views.i18n import set_language

urlpatterns = [
    path("i18n/setlang/", set_language, name="set_language"),
    # ...
]
```

Link from a Reflex page:

```python
def language_switcher() -> rx.Component:
    return rx.hstack(
        rx.link("English", href="/i18n/setlang/?language=en&next=/"),
        rx.link("العربية", href="/i18n/setlang/?language=ar&next=/"),
        rx.link("Français", href="/i18n/setlang/?language=fr&next=/"),
    )
```

The next page load (or the next event after refresh) uses the new session language.

## Lazy vs eager cheat sheet

| Where | Import |
|:---|:---|
| Component literals, `@page(title=...)` | `gettext_lazy as _` |
| Handler toasts, exceptions | `gettext as _` |
| Model `verbose_name` | `gettext_lazy` |

When unsure, use `gettext_lazy`.

## Skip locale middleware on events

Replace the default skip list if you truly do not need locale on WebSocket events:

```python
from reflex_django.event_handler import DEFAULT_EVENT_MIDDLEWARE_SKIP

REFLEX_DJANGO_EVENT_MIDDLEWARE_SKIP = (
    *DEFAULT_EVENT_MIDDLEWARE_SKIP,
    "django.middleware.locale.LocaleMiddleware",
)
```

## Common bumps

**Strings stay in English:** run `compilemessages` after editing `.po` files.

**Component strings never translate:** you used `gettext` instead of `gettext_lazy`.

**Wrong language active:** check `DjangoUserState.language` and the session/cookie chain.
